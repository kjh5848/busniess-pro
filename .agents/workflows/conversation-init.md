# 워크플로: 대화 초기화 및 프로젝트 컨텍스트 복원 (Conversation Initialization)

새 대화에서 사업계획서 관련 요청이 오거나 이전 작업을 이어받아야 할 때 가장 먼저 수행하는 워크플로입니다. (단독 실행 가능)

## Step 1: 프로젝트 확인 및 선별
1. 사업계획서 관련 작업 요청을 받으면 `projects/` 폴더를 확인합니다. (스크립트 실행 또는 `list_dir`)
2. **기존 프로젝트가 있는 경우:** 
   - 목록을 유저에게 표시합니다.
   - 이어서 작업할 프로젝트를 선택할 것인지, 새 프로젝트를 생성할 것인지 묻습니다.
3. **기존 프로젝트가 없는 경우:**
   - 즉시 신규 프로젝트 생성을 안내하고 필요한 정보를 요청합니다 (`init_project.py` 활용).

## Step 2: 컨텍스트 복원 절차
사용자가 특정 프로젝트를 선택하거나 단독으로 프로젝트명을 언급한 경우 다음 절차를 따릅니다:
1. **메타데이터 확인:** `projects/{프로젝트명}/project.json` 파일을 읽고 파일 내의 진행 상태(`completed_sections`)를 파악합니다.
2. **아이템 정보 복원:** `projects/{프로젝트명}/context/context-card.md`를 읽어 현재 창업 아이템의 핵심 컨텍스트를 다시 이해합니다.
3. **작업 위치 안내:** 완료된 부분과 다음으로 진행할 섹션을 나누어 유저에게 현황을 간략히 브리핑합니다.
4. **최소 로드 원칙 적용:** 필요한 해당 섹션에 대한 파일 한정으로만 불러들여 컨텍스트를 효율화합니다. (전체 파일 로드 금지)
