# 컨텍스트 관리 전략 (Context Management Strategy)

> 사업계획서 작성 에이전트에서 컨텍스트를 효율적으로 관리하는 기술 가이드

## 1. 핵심 원칙: 최소 고신호 원칙

**컨텍스트 창 = 희소자원**이다. 모든 것을 한번에 로드하지 않고,
현재 작업에 필요한 최소한의 고품질 정보만 선별 로드한다.

```
[BAD]  모든 V1~V9 컴포넌트 + 모든 텍스트 파일 동시 로드
[GOOD] 해당 섹션 V8/V9 컴포넌트 1개 + 해당 references 파일 1개만 로드
```

---

## 2. 계층적 Progressive Disclosure (단계적 로드)

### Level 0: 항상 사용 가능 (컨텍스트 비용 없음)
- `SKILL.md` 섹션 맵 → 어떤 컴포넌트/파일이 있는지 파악

### Level 1: 작업 시작 시 로드 (1회)
- `목차.md` → 공식 섹션 구조 확인
- 해당 섹션의 references 파일 1개 로드

### Level 2: 필요 시 로드 (조건부)
- 기존 추출 텍스트 (`extracted_1.txt` ~ `extracted_3.txt`) → 기존 내용 파악 시
- 최신 JSX 컴포넌트 (V8 또는 V9) → UI 생성/수정 시
- 웹 검색 결과 → 시장 데이터 최신화 시

### Level 3: 명시적 요청 시만 로드
- 구 버전 컴포넌트 (V1~V7) → 이전 버전 비교가 필요한 경우만
- 전체 사업계획서 통합 검토 시 모든 V9 컴포넌트

---

## 3. 섹션별 컨텍스트 격리 (Context Isolation)

각 섹션을 독립적으로 작업하여 컨텍스트 오염 방지:

| 작업 | 로드할 파일 | 로드하지 않을 파일 |
|------|------------|-----------------|
| 문제인식 작성 | `ProblemRecognitionV9.jsx`, `extracted_1.txt` | `GrowthV9.jsx`, `FeasibilityV9.jsx` |
| 실현가능성 작성 | `FeasibilityV9.jsx`, `extracted_2.txt` | `ProblemRecognitionV9.jsx`, `GrowthV9.jsx` |
| 성장전략 작성 | `GrowthV9.jsx`, `extracted_3.txt` | 다른 섹션 컴포넌트 |

---

## 4. 대화 간 상태 유지 전략

### 문제: 새 대화에서 이전 맥락 소실
예비창업패키지 사업계획서 작성은 수십 번의 대화에 걸쳐 진행됨.
매 대화마다 처음부터 설명하는 비효율 발생.

### 해결책: KI(Knowledge Item) 시스템 활용
AI 에이전트가 대화 종료 시 KI로 핵심 결정사항을 저장:
- 아이템명, 핵심 기술, 타겟 고객
- 결정된 섹션 내용 요약
- 다음 작업 예정 섹션

### 단기 대안: 대화 시작 시 컨텍스트 파일 직접 제공
사용자는 새 대화에서 다음을 제공하거나 참조 지시:
```
"이전 대화에서 작성한 FeasibilityV9.jsx를 기반으로 
성장전략 섹션 문제인식 부분을 작성해줘"
```

## 5. 웹 검색 컨텍스트 통합 전략

### 검색 우선순위
1. 정부/공공 기관 데이터 (가중치 높음): 통계청, KISA, 한국바이오협회, KCDC
2. 글로벌 리서치사: McKinsey, Deloitte, IQVIA, Gartner, IDC
3. 학술 논문: PubMed, Google Scholar (최근 3년 이내)
4. 언론 기사: 연합뉴스, 매일경제 (보조 자료로만)

### 검색 결과 통합 방법
```
[검색 결과] → [관련 수치 추출] → [출처 태그 부착] → [섹션에 삽입]
예: "글로벌 AI 신약개발 시장: $X억 (IQVIA, 2024), CAGR XX%"
```

### 컨텍스트 블로트(Context Bloat) 방지
- 검색 결과 원문 전체를 컨텍스트에 넣지 않음
- 필요한 수치와 출처만 요약 추출하여 사용
- 같은 주제 검색은 1~2회로 제한

## 6. 토큰 효율화 체크리스트

작업 시작 전 확인 사항:
- [ ] 필요한 섹션 references 파일 1개만 로드했는가?
- [ ] 최신 버전(V8/V9) 컴포넌트를 사용하는가? (구버전 불필요 로드 금지)
- [ ] 웹 검색 결과는 수치+출처만 요약했는가?
- [ ] 이전 작업 컴포넌트(다른 섹션)는 로드하지 않았는가?
