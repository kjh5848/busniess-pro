# Business Plan Writer Skill (사업계획서 작성 스킬)

이 스킬은 한국 **예비창업패키지** 사업계획서를 체계적으로 작성하기 위한 종합 가이드입니다.

## 핵심 원칙

1. **증거 기반 작성**: 모든 주요 주장은 출처가 있는 데이터로 뒷받침 (예: `* 출처: 통계청(2025)`)
2. **목차 정합성**: 반드시 `목차.md`를 먼저 확인하고 공식 섹션 구조를 따름 (`official-toc-mapper` 스킬 연계)
3. **HWP 스타일 UI**: 텍스트 작성 후 UI 요청 시 `hwp-styled-ui` 스킬 연계
4. **심사위원 관점**: 작성 후 `business-plan-evaluator` 스킬로 검토
5. **컨텍스트 효율화**: 섹션당 해당 references 파일 1개만 로드 (토큰 절약)

---

## 🗺️ 사업계획서 섹션 맵

| 섹션 | 컴포넌트 (최신 버전) | 주요 내용 |
|------|-------------------|---------|
| □ 창업아이템 개요(요약) | `SummaryV8` | 아이템 소개, 차별성, 개발단계, 목표시장, 이미지 |
| 1. 문제인식 | `ProblemRecognitionV8/V9` | 시장현황, Pain Point, 해결 필요성 |
| 2. 실현가능성 | `FeasibilityV8/V9` | 개발계획, 차별성, 사업비 집행 |
| 3. 성과목표 | `PerformanceV5/V6` | KPI 테이블, 마일스톤 |
| 4. 성장전략 | `GrowthV8/V9` | BM, 시장분석, 마케팅, 예산 |
| 5. 팀 구성 | `TeamV3` | 대표자, 팀원, 채용 계획 |

## 🚀 작업 흐름 (3단계)

### Step 1: 컨텍스트 수집
요청받은 섹션에 따라 **하나의** references 파일을 로드:

- **섹션 작성 방법이 필요한 경우** → `references/sections.md` 로드
- **평가 기준 확인이 필요한 경우** → `references/evaluation-criteria.md` 로드
- **프롬프트 템플릿이 필요한 경우** → `references/prompt-templates.md` 로드
- **컨텍스트 관리 전략이 필요한 경우** → `references/context-management.md` 로드

또한 반드시 확인:
- `목차.md` (공식 목차, 프로젝트 루트)
- 기존 추출 텍스트 (`extracted_1.txt` ~ `extracted_3.txt`, `extracted_toc.txt`)
- 최신 컴포넌트 파일 (해당 섹션의 V8/V9 JSX 파일)

### Step 2: 웹 검색으로 시장 데이터 보완
섹션 작성 전, 다음 데이터를 최신 웹 검색으로 확보:
- 시장 규모 및 성장률 (TAM/SAM/SOM)
- 경쟁사 현황 및 기술 트렌드
- 규제 환경 및 정책 동향
- 통계청, 한국바이오협회, IQVIA 등 공신력 있는 출처 우선

### Step 3: 섹션 초안 작성
`references/sections.md`의 섹션별 가이드를 따라 작성. 필수 요소:
- **핵심 메시지 박스** (섹션 상단 굵은 테두리 인용문)
- **증거 기반 데이터** (수치+출처)
- **개조식 문체** (불필요한 영단어 자제, 한국어 우선)
- **논리적 흐름**: 현황 → 문제 → 해결 → 효과

## 📐 UI 생성 시 규칙

텍스트 초안 완성 후 UI 요청이 있으면:
1. `hwp-styled-ui` 스킬의 **Option B (V3 데이터 기반)** 적용
2. 페르소나 랭킹 먼저 제시 후 사용자 선택
3. `official-toc-mapper` 스킬로 목차 정합성 확인
4. Recharts 차트는 모노크롬 유지 (흑백 톤)
