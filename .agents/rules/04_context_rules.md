# 04. 컨텍스트 관리 및 상태 머신 규칙 (Context & State Rules)

## 🟢 컨텍스트 관리 규칙

### R-12: 최소 로드 원칙
- 현재 작업 섹션 관련 파일만 로드 (다른 섹션 컴포넌트 동시 로드 금지)
- `docs/references/` 등 원본 참조 파일은 한 번에 1개만 주로 로드
- 구버전 컴포넌트 (V1~V7) 열람은 명시적 비교 통계 요청 시에만 허용

### R-13: 프로젝트 진행 상태 즉시 갱신
- 섹션 초안 완성 시 → `docs/sections/{섹션}.md` 파일에 즉시 저장
- UI 컴포넌트 생성 시 → 루트 경로 `project.json`의 `ui_components` 배열 구조 업데이트
- 섹션 완료 시 → 루트 경로 `project.json`의 `completed_sections` 상태 업데이트

---

## 📋 프로젝트 상태 머신

```text
[미시작] ──→ [진행중] ──→ [검토중] ──→ [완료]
                ↑                          |
                └──────── [수정중] ←───────┘
```

| 상태 | 정의 | 다음 행동 |
|------|------|---------|
| `미시작` | 문서 및 초안 없음 | 킥오프 시작 및 작성 진입 |
| `진행중` | 초안 작성 중 | 계속 작성 및 데이터 근거 보완 |
| `검토중` | 평가/체크리스트 점검 완료, 수정 대기 | 다중 페르소나 피드백 반영 |
| `완료` | UI 포함 최종 완성 파이프 | 다음 작성 섹션으로 이동 |
| `수정중` | 완료 후 재수정 요청 | 해당 섹션 파일 부분만 수정 |
