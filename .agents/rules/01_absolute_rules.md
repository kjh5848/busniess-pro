# 01. 절대 규칙 (Absolute Rules)

> 이 파일의 규칙은 모든 스킬·워크플로보다 **우선 적용**됩니다.

## 🔴 절대 규칙 (절대 어기면 안 됨)

### R-01: 프로젝트 경로 원칙 (Git 브랜치 기반)
- 더 이상 별도의 `projects/{아이템명}/` 폴더를 사용하지 않으며, **작업 공간의 루트 폴더를 기준으로 Git 브랜치를 통해 각각의 사업계획서 버전을 관리**합니다.
- 모든 컨텍스트와 메타데이터는 오직 현재 워크스페이스 루트를 기준으로 찾고 생성합니다.

### R-02: 출처 없는 수치 금지
- 모든 시장 규모, CAGR, 통계 수치는 반드시 출처 태그 포함.
  - 형식: `* 출처: 기관명(연도)`
- 출처를 찾을 수 없는 수치는 작성하지 않고 사용자에게 확인 요청.

### R-03: 목차 정합성 의무
- UI 컴포넌트 생성 전 문서 공식 목차(`docs/references/toc.md` 등) 또는 사용자 제공 목차를 반드시 확인.
- 공식 목차와 1:1 매핑이 되지 않는 섹션은 생성 금지 → 사용자에게 먼저 보고.

### R-04: 사업비 집행 규정 준수
- 사업비 관련 내용 작성 시 `fund-execution-guidelines` 스킬 기준 반드시 적용.
- 인건비, 개인 용도 물품 항목 절대 포함 불가.

### R-05: 파일 저장 위치 규정 (루트 구조 평탄화)
| 파일 유형 | 저장 위치 |
|---------|---------|
| 섹션 텍스트 초안 | `docs/sections/{식별자}.md` |
| 시장 조사 데이터 | `docs/context/market-data.md` |
| 참조 문서 (HWP 등) | `docs/references/` |
| 프로젝트 메타데이터 | 루트 최상단 `project.json` |
| React UI 컴포넌트 | `business-plan-ui/src/features/{식별자}/` 또는 `business-plan-ui/src/components/ui/` |
