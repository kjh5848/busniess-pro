# 커스텀 스킬 및 워크플로우 한글 요약 (Korean Skill Summary)

현재 에이전트에 등록되어 있는 스킬(Skills)과 워크플로우(Workflows)들이 각각 어떤 역할을 하는지 알기 쉽게 정리한 한글 가이드 문서입니다.
프롬프트나 지침의 원본은 영어로 되어 있더라도, 각 스킬은 아래와 같은 목적을 가지고 실행됩니다.

## 🛠️ 스킬 목록 (Skills)

1. **`hwp-extractor` (HWP 텍스트 추출기)**
   * **역할**: 파이썬(`olefile` 등)을 사용하여 한글(`.hwp`) 문서 내부의 순수 텍스트를 안전하게 추출하는 가이드입니다.
   * **사용처**: 사용자가 HWP 파일을 업로드하고 분석이나 요약을 요청할 때, 혹은 HWP 문서를 바탕으로 웹 UI를 생성하기 전에 문서 내용을 읽어들이기 위해 사용합니다.

2. **`hwp-styled-ui` (통합 HWP 및 데이터 UI 스타일)**
   * **역할**: 한글(HWP) 문서의 시각적 느낌(흑백 톤, 각진 모서리, 기본 표 스타일)을 모방한 심플한 React UI와, 데이터 차트(그래프), 구체적인 예산 표, 인용구 등을 포함한 고도화된(V3) 사업계획서 화면을 모두 생성할 수 있는 통합 UI 가이드입니다.
   * **사용처**: 단순한 테이블이나 양식을 베낄 때는 내장된 `hwp-theme.css` 자산을 참고하고, "문제인식"이나 "성장전략"처럼 기획과 데이터를 그럴싸하게 시각화해야 하는 페이지를 만들 땐 내장된 `ExampleV3.jsx` 자산을 참고해 화면을 만듭니다.

3. **`official-toc-mapper` (공식 목차 매퍼)**
   * **역할**: AI가 UI를 생성할 때 원본 문서의 '공식 목차(`목차.md` 등)' 구조를 마음대로 요약하거나 빠뜨리지 않고 **1:1로 엄격하게 반영**하도록 강제하는 가이드입니다.
   * **사용처**: 공식 양식(예: 1-1, 1-2, 1-3)의 항목이 하나라도 누락되면 안 되는 사업계획서 템플릿 컴포넌트를 생성할 때 함께 적용합니다.

4. **`modern-web-ui` (모던 웹 UI)**
   * **역할**: 화려한 색상, 그라데이션, 애니메이션 효과, 그림자 등을 포함하는 현대적이고 프리미엄급 웹 UI를 만드는 가이드입니다.
   * **사용처**: HWP 스타일이 아닌 "스타트업 랜딩 페이지", "관리자 대시보드" 등 세련된 사용자 대상(B2C/B2B) 웹앱 화면을 만들 때 사용합니다.

5. **`skill-creator` (스킬 생성 가이드)**
   * **역할**: AI가 새로운 스킬을 만들거나 기존 스킬을 개선할 때 지켜야 하는 원칙(간결성 유지, 리소스 분리, 점진적 개방 원칙 등)을 지정한 코어 지침서입니다.

---

## 🚀 워크플로우 (Workflows)

1. **`hwp-to-react-ui` (HWP -> React UI 변환 워크플로우)**
   * **역할**: 사용자가 HWP 문서를 던져주었을 때 이를 분석하고 화면으로 만들어내는 전체 순서를 정의한 파이프라인(순서도)입니다.
   * **동작 순서**: 
     1. `hwp-extractor`로 텍스트 추출
     2. `official-toc-mapper`로 목차 정리
     3. 사용자 요구(기초 vs 고급데이터)에 따라 `hwp-styled-ui` 내부 에셋(CSS/JSX) 로드
     4. React 컴포넌트 생성 및 반영
     5. 브라우저로 렌더링 확인 후 사용자에게 안내
